Oasis API¶

Summary¶

Note

The Oasis REST API currently supports running model analyses. We are extending the API to include exposure management capabilities.

A typical interaction with the Oasis REST API is illustrated in the following diagram. Exposure is uploaded to the platform, an analysis is started, the analysis status is polled, and when the anlaysis completes the outputs are downloaded.

DELETE /analysis_status¶

Delete an analysis status resource — description: Deletes an analysis status resource. If no location

is given all analysis status resources will be deleted.

description: OK
404:
description: Resource not found

in: path description: The location of the analysis status resource to delete. required: true type: string

The tar file can be compressed or uncompressed.

produces: - application/json responses:

description: The exposure summary of the created exposure resource. schema:

Analysis Settings Schema¶

The following schema defines the analysis settings document that is used to configure an analysis. This document defines: * General analysis settings, such as the number of Monte Carlo samples. * Specific model settings, such as running/not-running specific sub-perils. * What outputs to generate, for what financial perspectives and by what reporting groups.
