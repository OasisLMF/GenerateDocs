src.model_execution_worker.tasks.on_error(request, ex, traceback, record_task_name, analysis_pk, initiator_pk)[source]¶

Because of how celery works we need to include a celery task registered in the current app to pass to the link_error function on a chain.

This function takes the error and passes it on back to the server so that it can store the info on the analysis.

src.model_execution_worker.tasks.prepare_complex_model_file_inputs(complex_model_files, upload_directory, run_directory)[source]¶

Places the specified complex model files in the run_directory.

The unique upload filenames are converted back to the original upload names, so that the names match any input configuration file.

On Linux, the files are symlinked, whereas on Windows the files are simply copied.

Args:
complex_model_files (list of complex_model_data_file): List of dicts giving the files

to make available.

upload_directory (str): Source directory containing the uploaded files with unique filenames. run_directory (str): Model inputs directory to place the files in.

Returns:

None.
